## 2. Sign up for Render

Go to https://render.com ŌåÆ **Sign Up** ŌåÆ "Sign up with GitHub". Render asks

## 3. Create the Web Service

In the Render dashboard:

1. Click **+ New** ŌåÆ **Web Service**.
2. Pick the `wedding-rsvp` repo you just pushed. Click **Connect**.
3. Render reads your `render.yaml` and pre-fills most settings. Verify:
   - **Name**: `wedding-confirmation` (or anything ŌĆö this becomes part of the URL)
   - **Runtime**: `Python`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 60`
   - **Instance Type**: `Free`

> ŌÜĀ’ĖÅ If you pushed only the `wedding_rsvp` folder as the repo root, you can
> ignore `rootDir`. If you pushed the whole `wed/` folder, make sure the
> **Root Directory** field is set to `wedding_rsvp`.

## 4. Set environment variables

Still on the "Create Web Service" page, scroll to **Environment Variables**.
You need at least two:

| Key             | Value                                                |
|-----------------|------------------------------------------------------|
| `ADMIN_PASSWORD` | A strong password only you know (you'll use this to log in to `/admin`) |
| `SECRET_KEY`    | Will be auto-generated by Render ŌĆö leave as-is       |

**Optional** (only if you want email notifications when guests confirm):

| Key            | Value                                                |
|----------------|------------------------------------------------------|
| `SMTP_HOST`    | `smtp.gmail.com`                                     |
| `SMTP_PORT`    | `587`                                                |
| `SMTP_USER`    | `jmussamuhindo@gmail.com`                            |
| `SMTP_PASS`    | A **Gmail App Password** (NOT your normal password ŌĆö see below) |
| `NOTIFY_EMAIL` | `jmussamuhindo@gmail.com`                            |

**Getting a Gmail App Password:**
1. Google Account ŌåÆ Security ŌåÆ turn on **2-Step Verification** if not on.
2. Then go to https://myaccount.google.com/apppasswords
3. Generate a password named "Wedding RSVP". Copy the 16-character code.
4. Paste it as the `SMTP_PASS` value on Render.

## ŌÜĀ’ĖÅ Important: Free-tier limitations

Render's **free** plan has two things to know about:

### 1. The site sleeps after 15 minutes of no traffic
First visit after a sleep takes ~30 seconds (cold start). After that
it's instant. Guests filling out the form rarely notice. To eliminate
this, upgrade to **Starter** ($7/month).

### 2. The filesystem is wiped on every redeploy
Every time you push new code or Render redeploys, `responses.csv` is
reset to empty. **For your wedding, you have three options:**

**Option A ŌĆö Don't push code changes once guests start submitting.**
Once the URL is shared, leave the app alone. Render won't auto-redeploy.
Click **Export CSV** in the admin dashboard regularly to keep a backup.

**Option B ŌĆö Upgrade to Starter ($7/month) and add a persistent disk.**
In `render.yaml`, uncomment the `disk:` block at the bottom, change
`plan: free` to `plan: starter`, commit and push. Render will mount a
1 GB disk at `/var/data`. Then update `app.py`:

```python
# Change this line:
RESPONSES_FILE = BASE_DIR / "responses.csv"
# To this:
RESPONSES_FILE = Path(os.environ.get("DATA_DIR", BASE_DIR)) / "responses.csv"
```

And add an env var `DATA_DIR=/var/data` in Render.

**Option C ŌĆö Export the CSV every day.** Just visit `/admin` and click
**Ō¼ć Export CSV** once a day. Keep the file on your computer.

For a one-time event, **Option A or C** is usually fine.

## ŌØō Troubleshooting

**Build fails with "ModuleNotFoundError":** check that `requirements.txt`
is committed to git (`git ls-files | grep requirements.txt`).

**App boots but `/admin` won't accept the password:** double-check the
`ADMIN_PASSWORD` env var on Render. After changing env vars, click
**Manual Deploy ŌåÆ Deploy latest commit**.

**Guests get 500 errors:** click the **Logs** tab in Render to see what
went wrong.

**Cold-start delay is annoying:** upgrade to Starter ($7/mo) ŌĆö never sleeps.
